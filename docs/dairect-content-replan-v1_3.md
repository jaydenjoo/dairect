## 1. 진단 (요약)

dairect.kr은 시각적으로 완성도가 높지만 **"고급 디자인 스튜디오"** 로 읽힙니다. 정작 잡으려는 고객은 **"이번 주에 빨리 만들어줄 누군가를 급하게 찾는 SMB"** 인데, 이 사람들이 사이트를 보면 "여긴 비싸 보이고 천천히 만드는 데구나"라고 오해합니다.

격차의 본질은 **디자인이 아니라 메시지**이며, v1.3의 핵심 통찰은:

> *"못 하는 것을 정직하게 공개하는 것이, 할 수 있는 것을 강조하는 것보다 신뢰를 더 만든다."*
>
> *"정직한 카운팅(라이브 4 / 실험 8)이 부풀린 12보다 강하다."*
>
> *"자동화된 슬롯 표시는 시스템적 차별점이다 — SI/프리랜서가 못 하는 것."*

## 2. 메시지 아키텍처 (3-Tier)

```
Tier 1 (10초)  →  비교 컨텍스트 (시간) + AI 진입 장벽 해소 + 라이브 슬롯
Tier 2 (1분)   →  3 페르소나 + 4 패키지 + 못하는 것 + 도전 중인 것
Tier 3 (3분+)  →  검증된 실적(라이브 4) + Founder + Contact
```

### 3-9. 4 Moves — Ship & Handoff에 사업화 동행 흡수 (수정)

### 3-10. (수정) 30일 → 14일 무상 운영 자문 + 명확한 경계

**v1.2 → v1.3 변경:** 30일 → 14일, 경계 5개 명시.

**이유:** 동시 진행 3건 시 자문만으로 주당 15~40시간 잠식 위험. Build/Scale 클라이언트가 늘수록 자문 부담이 신규 프로젝트 시간을 잠식.

· 출시 후 14일간 텍스트 자문 (슬랙 1개 채널)

· 건당 30분 이내 답변, 월 5회까지
· 코드 수정·기능 추가는 별도 견적

왜 14일인가
· 출시 후 첫 2주가 90% 이슈 발생 구간
· 14일 후 = 자연스러운 유료 유지보수 전환 시점
```

```
Scale 패키지 — 90일 파트너십 (변경 없음)

· 90일간 우선 응답 + 월간 리뷰
· 첫 매출까지 마케팅·운영 자문 포함
· 별도 SLA는 계약 시 명시

> 💡 **전략 노트:** "30일 무상 무제한"은 약속이 모호해서 양쪽 모두 불만. "14일 + 24h SLA + 월 5회 + 슬랙 1채널 + 30분 이내"는 약속이 명확해서 양쪽 모두 만족. **모호한 무제한 < 명확한 한정** — 이게 일반 프리랜서/SI와 또 다른 차별점.
