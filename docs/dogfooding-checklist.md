## 🎯 dogfooding의 의미

1. **"내가 만든 도구를 내가 직접 써서 제품을 증명한다"** — 판매 전 품질 게이트
2. **Paper 스펙과 Real UX 차이 발견** — 문서에서 놓친 실제 불편 포착
3. **2차 확장 우선순위 수집** — "직접 써보니 이게 더 필요하더라"
4. **SI 수주 신뢰 자산** — "저는 이 도구로 직접 제 프로젝트 관리합니다" 시연 가능

## 🔄 End-to-End 프로젝트 생명주기 검증

**목표**: 실제 프로젝트 1건 이상을 **리드 → 수금**까지 Dairect 안에서 전체 관리 완료.

### 단계 1: 리드 생성 (신규 문의 발생 시)
- [ ] `/dashboard/leads` → 리드 신규 생성
- [ ] 예산 / 마감 기한 / 요구사항 입력
- [ ] 상태: 신규 → 진행 → 성공/실패 단계 이동 경험
- [ ] 🔍 확인: 파이프라인 시각화 정확한가? 상태 전환 UX 자연스러운가?

### 단계 2: 고객 등록
- [ ] `/dashboard/clients` → 신규 고객 등록
- [ ] 사업자번호 / 담당자 / 연락처 입력
- [ ] 🔍 확인: 중복 등록 방어되는가? 입력 검증 친절한가?

### 단계 3: 프로젝트 생성
- [ ] `/dashboard/projects` → 신규 프로젝트 생성 + 고객 연결
- [ ] 예상 기간 / 금액 설정
- [ ] 마일스톤 3~5개 생성 (예: 기획, 디자인, 개발, 배포)
- [ ] 🔍 확인: 마일스톤별 체크포인트 명확한가? 금액 분할(착수금/중도금/잔금) 연결?

### 단계 4: 견적서 작성 (수동 + AI)
- [ ] **수동 견적 1건**:
  - [ ] `/dashboard/estimates/new` → 항목 직접 입력
  - [ ] 착수금/중도금/잔금 비율 확인
  - [ ] PDF 생성 → 내용 확인 → 고객에게 전달 (이메일/카톡)
- [ ] **AI 견적 1건**:
  - [ ] 요구사항 텍스트 입력 → AI 견적 생성 버튼
  - [ ] AI 출력 결과 검토 (항목/인일수/난이도/카테고리)
  - [ ] 수동 수정 후 확정 저장
  - [ ] 🔍 확인: AI 출력 품질이 실제 업무에 유용한가? 수정 비율은?

### 단계 5: 계약서 작성
- [ ] 계약서 표준 템플릿 선택
- [ ] 변수 자동 매핑(프로젝트/고객/견적 정보) 확인
- [ ] 편집 가능 영역(하자보수 기간, 지적재산권, 손해배상 한도) 조정
- [ ] PDF 생성 → 고객 전달
- [ ] 🔍 확인: 한국 SI 표준 조항 누락 없는가? 법적 검토 필요 항목 식별

### 단계 6: 마일스톤 진행 + 고객 포털 공유
- [ ] `/dashboard/projects/[id]` 마일스톤 상태 업데이트 (진행 중 → 완료)
- [ ] **고객 포털 토큰 생성** → URL 복사 → 고객에게 전달
- [ ] 고객 측에서 `/portal/[token]` 접근 → 어떤 정보 노출되는지 확인
- [ ] 🔍 확인: 포털에서 비공개 정보(금액/다른 프로젝트/내부 메모) 누출 0건?
- [ ] 고객이 피드백 남기면 `/dashboard/projects/[id]` 피드백 탭에서 확인

### 단계 7: 청구서 발행 + 수금
- [ ] `/dashboard/invoices/new` → 마일스톤 연결 청구서 생성
- [ ] 세금계산서 정보 확인 (공급자/공급받는자)
- [ ] PDF 생성 → 고객에게 발송
- [ ] 수금 확인 시 "수금 완료" 상태 변경
- [ ] 🔍 확인: 청구서 번호 채번(INV-YYYY-NNN) 정상? 부가세 계산 정확?

### 단계 8: 프로젝트 종료
- [ ] 모든 마일스톤 완료 + 수금 확인
- [ ] 프로젝트 상태 "완료"로 전환
- [ ] 🔍 확인: 완료 프로젝트가 포트폴리오에 자동 반영되는가?

## 📝 이슈 기록 양식

실사용 중 발견된 문제를 기록. GitHub Issues 생성 또는 별도 파일.

### 이슈 템플릿
```markdown
## [우선순위] 제목

- **발견 시점**: 2026-MM-DD, 어떤 작업 중
- **재현 경로**: 1) /dashboard/X → 2) Y 버튼 클릭 → 3) 오류
- **기대 동작**: 무엇이 일어나야 했는가
- **실제 동작**: 무엇이 일어났는가 (스크린샷 첨부 권장)
- **영향 범위**: Jayden만 / 모든 사용자 / 데이터 손실 위험
- **추정 원인**: (알면 메모, 모르면 skip)
- **관련 파일**: src/...
```

### 우선순위 분류

| 등급 | 기준 | 대응 |
|---|---|---|
| 🔴 **High** | 기능 자체가 동작 안 함 / 데이터 손실 / 보안 | 즉시 수정 |
| 🟡 **Medium** | 기능은 동작하나 UX 불편 / 성능 이슈 | 1주 내 수정 |
| 🟢 **Low** | 디자인 미세 조정 / nice-to-have | 2차 검토 시 반영 |
